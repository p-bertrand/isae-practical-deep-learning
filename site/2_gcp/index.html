    Installation GCP¶

    Création et démarrage de l'instance Google Cloud Platform

    https://console.cloud.google.com/

    0 - Pré installation¶

    Vous devriez normalement avoir google cloud d'installé sur votre machine (et avoir déjà utilisé gcloud)

    1 - Se connecter à Google Cloud Shell¶

    2 - Créer une instance Google Cloud Platform¶

    IMAGE_FAMILY="tf-latest-gpu"
ZONE="europe-west1-d"
INSTANCE_NAME="tp-isae-dev-gpu" # Vous pouvez renommer votre machine

gcloud compute instances create ${INSTANCE_NAME} \
  --zone=${ZONE} \
  --image-family=${IMAGE_FAMILY} \
  --image-project=deeplearning-platform-release \
  --machine-type=n1-standard-8 \
  --maintenance-policy=TERMINATE \
  --preemptible \
  --tags=jupyter \
  --scopes default \
  --scopes storage-rw \
  --accelerator='type=nvidia-tesla-k80,count=1' \
  --metadata='install-nvidia-driver=True'

    Si GCP refuse la création d'une instance pour manque de ressources, changez la zone (voir ici https://cloud.google.com/compute/docs/gpus/#introduction)

    Si vous vous rendez sur https://console.cloud.google.com/compute/instances vous devriez voir votre instance en cours de création. Une fois celle-ci créée, ceci s'affiche

    NAME             ZONE            MACHINE_TYPE   PREEMPTIBLE  INTERNAL_IP  EXTERNAL_IP    STATUS
tp-isae-dev-gpu  europe-west1-d  n1-standard-8  true         10.132.0.11  35.233.87.123  RUNNING

    3 - Faire un tunnel ssh depuis votre machine vers l'instance¶

    Normalement cela devrait marcher en 4G ou sur ISAE-EDU

    Depuis un terminal de votre machine (pas google cloud shell) où le gcloud SDK est installé:

    gcloud compute ssh --zone europe-west1-d (votre-nom-utilisateur-gcp)@tp-isae-dev-gpu -- -L localhost:8080:localhost:8080

    4 - Ouvrir votre navigateur internet et se connecter à l'instance¶

    http://localhost:8080/

    5 - Lancer un jupyter python3 puis rentrer au moins ces commandes:¶

    %env TP_ISAE_DATA= ./data/`
!pip3 install git+https://github.com/fchouteau/isae-practical-deep-learning.git#egg=khumeia\&subdirectory=src

    Une fois le package pip installé, vous pouvez supprimer la cellule (ou démarrer un autre notebook) Puis 

    import khumeia

khumeia.download_train_data()

    Vous devriez voir un dossier data/ créé à la racine ainsi que les logs suivants

    4bis plan B si ça ne marche pas¶

    Directement depuis google cloud shell

    Depuis google cloud shell gcloud compute ssh --zone europe-west1-d $USER@tp-isae-dev-gpu -- -L 8080:localhost:8080

    Ensuite "open web preview on port 8080"
